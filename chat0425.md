# 板块主线追踪器设计方案 — 实战评审意见

我从一个做过多年量化择时和板块轮动策略的实战视角，给你逐条点评。先说**总体判断**，再针对你的每个回答给出**具体修改建议**，最后指出**几个被你忽略但很关键的坑**。

## 〇、总体评价（先看结论）

**优点**：
- 工作流分层清晰（大盘→板块→龙头），符合"自上而下"的实战逻辑
- 四维评分（动量+资金+情绪+结构）覆盖了主线识别的核心要素
- 与红利 Skill 形成"攻防互补"的定位很好

**主要风险（按严重程度排序）**：

1. 🔴 **"指南针活跃市值"作为唯一核心信号，存在严重的单点依赖和数据可得性问题** —— 详见第一节
2. 🔴 **"第一个放量涨停"作为龙头标准，在实战中会大量误判** —— 详见龙头部分
3. 🟡 **每日运行 + 周度持续性判断的频率不匹配**
4. 🟡 **概念板块300+个全扫，噪音会淹没信号**
5. 🟡 **缺少"主线切换"的识别机制**（旧主线见顶 vs 新主线启动）

## 一、关于"指南针活跃市值"——必须重新讨论

### 1.1 这是什么指标？（基于公开信息的理解，置信度中等）

指南针软件（北京指南针科技，股票代码 300803）的"活跃市值"是其付费版本的**专有指标**，大致逻辑是：

> 对每只个股按"活跃度"（换手率、成交额、振幅等）加权，再乘以流通市值，汇总得到全市场的"参与博弈的有效市值"。

**核心争议点**：
- ❌ **无公开 API、无官方数据导出接口**——这是付费软件的核心卖点之一
- ❌ AKShare、Tushare、东财公开接口**均无此指标**
- ⚠️ 即使你有指南针账号，软件也不提供历史数据批量导出（只能看当日）
- ⚠️ "单日或2日涨幅≥4%"这个阈值，**在公开资料中我没有找到指南针官方的回测依据**——这可能是某个营销课程或社群提出的经验法则，需要你自己核实出处和样本

> 💡 **置信度声明**：以上判断基于我对市面金融软件生态的了解。我建议你自己用指南针软件验证："活跃市值"是否真的可以导出历史数据；以及"4%阈值"的原始出处。

### 1.2 实战建议：不要把它作为"唯一核心信号"

**问题**：你把一个**无法稳定获取、无公开验证、阈值来源不明**的指标作为系统的核心，整个 Skill 的可执行性会被卡死。

**强烈建议改为方案 C（公开数据近似复刻）+ 多信号确认**：

```python
# 自建"活跃市值代理指标" - 全A股口径
活跃市值代理 = Σ(个股流通市值 × 个股换手率)
            = 全市场成交额 (近似)

# 更精细版：剔除一字板和ST，只算真正"博弈中的市值"
有效活跃市值 = Σ(流通市值 × 换手率) for 股票 in {振幅>2% 且 非ST 且 非一字板}
```

**择时信号改为多重确认（任一触发即可）**：

| 信号 | 阈值 | 含义 |
|---|---|---|
| 全A成交额单日涨幅 | ≥ 25% | 资金大规模入场 |
| 全A成交额 2日累计涨幅 | ≥ 40% | 持续性放量 |
| 自建"有效活跃市值"涨幅 | 单日≥4% 或 2日≥4% | 复刻指南针逻辑 |
| 沪深300在20日线上 + 斜率向上 | 持续≥3天 | 趋势确认 |
| 涨停家数 / 跌停家数 | ≥ 5 | 情绪面多头 |

**输出分档**（替代单一"是/否"）：

- **强多头**：3个以上信号同时触发
- **温和多头**：2个信号触发
- **震荡**：1个信号
- **空头**：0信号 + 大盘破位

这样既保留了你想要的"活跃市值"思路，又不依赖无法获取的数据。

## 二、关于"第一个放量涨停=龙头"——实战陷阱

你在问题5的回答："**板块、行业内第一个放量涨停的票**"

### 2.1 为什么这个标准在实战中会大量误判？

我做板块轮动这些年，**"首板≠龙头"是血的教训**。具体场景：

| 场景 | 问题 |
|---|---|
| **诱多首板**：板块异动初期，某只小票被游资"试盘"涨停，次日板块整体跟不上，首板烂板 | 系统会把它标为龙头，但它实际是"试盘货" |
| **题材纯度低**：板块内蹭概念的边缘股先涨停（业绩烂、市值小），真龙头反而后启动 | 系统识别错龙头方向 |
| **机构票vs游资票混淆**：机构主线（如新能源）启动慢、不靠涨停；游资主线（如AI）首板乱飞 | 单一标准无法兼容两类主线 |
| **早盘竞价涨停**：开盘就一字板的票，根本不是"放量"涨停 | 需要区分开盘方式 |

### 2.2 改进方案：分类型识别龙头

```
龙头识别 = 主线类型判断 + 对应规则

【判断主线类型】
- 板块内涨停股平均市值 > 200亿 → 机构主线
- 板块内涨停股平均市值 < 100亿 + 连板股多 → 游资主线
- 中间值 → 混合主线

【机构主线龙头】
- 近20日涨幅Top3
- 成交额持续放大（5日均量 > 20日均量1.5倍）
- 市值在板块前30%
- 不强求涨停

【游资主线龙头】
- 板块启动日（板块涨幅首次>3%且后续延续）首个封板的票
- 且当日封单金额 > 流通市值的1%
- 且次日不烂板（开盘价 > 前日涨停价×0.97）
- 连板高度领先
```

> 💡 **强烈建议**：龙头判断不要在第一版就追求精准，先输出**"龙头候选名单 Top5 + 关键标签"**（连板数、市值、首板日期、近5日资金净流入），把判断权留给用户。这比"算法武断指认龙头"更安全。

## 三、对你其他回答的具体点评

### 问题2（行业+概念全扫）— 建议改为 C

**问题**：同花顺概念板块约 350+ 个，且存在大量**重复归类**（一只票被分到 10+ 个概念）。全扫会导致：
- 同一波资金被多个概念重复计数
- 噪音概念（如"昨日涨停"、"次新股"这类技术性概念）干扰排名
- 运行时间从 ~30秒 变成 ~5分钟

**建议**：
- 行业板块全扫（76个，必扫）
- 概念板块**预过滤**：剔除技术性概念（含"涨停"、"次新"、"破净"等关键词），只保留"题材性概念"，再取近20日涨幅Top50

### 问题3（每日运行） — 没问题，但需要调整持续性逻辑

**与你问题4的回答冲突**：每日运行 + 读取"上周JSON" → 频率不一致。

**建议**：
- **滚动窗口**：维护最近 30 天的每日 JSON
- **持续性判断**：某板块"在主线候选名单中出现 ≥ 5/10 个交易日" → 升级为确认主线
- 这样既符合每日运行节奏，又能识别新老主线切换

### 问题4 — 强烈建议升级为 SQLite

每日运行场景下，30 天 = 30 个 JSON 文件，再加上每个板块的成分股快照、龙头跟踪记录……JSON 文件管理会很快失控。

**SQLite 的好处**：
- 单文件，无服务依赖
- 标准 SQL 查询历史
- ~50 行代码就能搞定 schema

> 复杂度增加非常有限，但 6 个月后你会感谢自己。

### 问题6（预留联动接口） — 赞成，但要明确接口

建议两个 Skill 共享一份 `market_regime.json`：

```json
{
  "date": "2026-04-25",
  "regime": "strong_bull",  // strong_bull / mild_bull / range / bear
  "signals_triggered": ["volume_surge", "ma20_up", "limit_up_ratio"],
  "leading_sectors": ["半导体", "AI算力"],
  "defensive_sectors_recommend_weight": 0.3  // 红利组合可读取此字段调整仓位
}
```

## 四、被你忽略的几个关键问题

### 4.1 主线"见顶"识别（最重要的缺失）

实战中，**识别主线启动的难度，远小于识别主线见顶**。建议增加：

```
主线见顶预警信号：
- 板块内连板股数量连续2日下降
- 龙头股出现"放量天量长上影/炸板"
- 板块20日涨幅 > 40% 且 5日涨幅 < 0
- 资金从净流入转为净流出连续3日
- 出现"高标接力失败"（最高板炸板后无新高板）
```

主线见顶往往就是 1-2 天的事，每日运行的最大价值就是**及时预警**。

### 4.2 "伪主线"过滤

需要剔除：
- **政策一日游**：当天爆炒、次日就熄火的概念
- **个股带动**：板块涨幅完全由 1-2 只大权重股贡献（如某个板块只因为茅台涨而涨）
  - 检测：板块涨幅 vs 板块内**等权涨幅**的差异；如果差异 > 50%，标记为"伪板块效应"

### 4.3 数据时效性陷阱

- AKShare 的同花顺接口**经常被反爬**（403错误、接口变更）
- 板块资金流数据通常是**收盘后 30 分钟**才更新
- 涨停池数据**盘中实时变动**，需要明确取数时点（建议固定 15:30 后取数）

**建议**：每个数据源都加 fallback，比如同花顺接口失败时切换到东财接口。

### 4.4 回测验证（最关键）

任何择时/选股策略，**没有回测就是裸奔**。

建议在 v1 之后立刻做：
- 用 2020-2025 的历史数据回测
- 检验"主线确认信号"在历史上的胜率（主线确认后 5/10/20 日的板块超额收益）
- 检验"主线见顶信号"的提前性
- 至少要有 50+ 次历史样本，胜率 > 55% 才算可用

> ⚠️ 没有回测验证的择时系统，再漂亮的报告也只是"事后诸葛亮"。

## 六、给你（领域新人）的几句心里话

1. **板块主线追踪的本质是"概率游戏"**，不是预测。任何 Skill 输出都应该是"概率指引"而非"行动指令"。

2. **不要迷信任何单一指标**——包括指南针的活跃市值。任何"圣杯指标"在公开后都会失效，何况它本身的有效性都缺乏公开证据。

3. **顺势而为 ≠ 追高**。板块主线 Skill 在多头中段最有效，在大盘启动初期容易误判（信号未成形），在大盘见顶时最危险（主线诱多）。**Skill 必须告诉用户当前处于哪个阶段**。

4. **先把"减少错误"做好，再追求"抓住大牛股"**。第一版不要追求性能，追求**不误判**。

5. **回测、回测、回测**。我见过太多漂亮的策略文档，跑回测一看胜率 40%。
