## 一、背景与定位

### 1.1 为什么需要这个 Skill？

当前项目中已有：
- `dividend-portfolio-tracker`：红利组合防守，长期收息
- `maotai-dividend-tracker`：单只深度分析

**缺失的能力**：当 A 股进入**多头区间**时，红利策略虽然稳健，但无法捕捉**板块主线**带来的超额收益。板块主线识别是"顺势进攻"工具，与红利策略形成互补。

### 1.2 核心目标

输入：当前市场状态（全自动数据抓取）  
输出：
1. 当前是否处于多头区间？
2. 如果是，**谁是主线板块**？
3. 主线的**龙头股**是谁？
4. 主线的**持续性**如何？（需历史对比）

---

## 二、核心问题定义

| 问题 | 定义 |
|------|------|
| **什么是"多头区间"？** | 大盘（上证/沪深300）处于中期上升趋势：指数在 20 日均线上方 + 均线斜率向上 + 成交量温和放大 |
| **什么是"板块主线"？** | 在多头区间内，**持续跑赢大盘**、**资金持续流入**、**有板块效应**（多股联动涨停/连板）的领涨板块 |
| **什么是"龙头股"？** | 主线板块内涨幅领先、市场辨识度最高、带动板块情绪的个股 |

## 三、数据来源

基于 AKShare，已有接口：

| 数据 | 接口 | 用途 |
|------|------|------|
| 行业板块列表 | `stock_board_industry_name_ths` | 获取全部行业板块 |
| 行业板块历史行情 | `stock_board_industry_hist_ths` | 计算板块涨跌幅、均线 |
| 概念板块列表 | `stock_board_concept_name_ths` | 获取热点概念 |
| 概念板块历史行情 | `stock_board_concept_hist_ths` | 计算概念涨跌幅 |
| 板块资金流向 | `stock_fund_flow_industry` / `stock_fund_flow_concept` | 识别资金主攻方向 |
| 板块成分股 | `stock_board_industry_cons_ths` / `stock_board_concept_cons_ths` | 找出板块内龙头股 |
| 涨停池 | `stock_zt_pool_em` | 统计板块涨停股数量、连板高度 |
| 大盘指数 | `stock_zh_index_daily` | 判断多头区间 |

## 四、核心工作流

```
Step 1: 大盘环境判断
    └─ 上证/沪深300 是否在20日均线上方？均线斜率是否向上？
    └─ 输出：是/否处于多头区间；若是，多头持续天数

Step 2: 板块初筛（全市场行业+概念）
    └─ 计算近20日涨跌幅 → 筛选涨幅前20%的板块
    └─ 计算近5日涨跌幅 → 筛选短期加速的板块
    └─ 叠加资金流向 → 剔除"无量上涨"的板块

Step 3: 板块深度评分（四维100分制）
    ├─ 动量维度(30分)：20日涨幅排名、5日加速、相对大盘超额收益
    ├─ 资金维度(30分)：近5日/20日净流入金额、净流入占比
    ├─ 情绪维度(25分)：板块内涨停股数量、连板高度、涨停股占比
    └─ 结构维度(15分)：板块指数是否在20日均线上方、量价配合

Step 4: 主线确认
    └─ 综合得分≥75分的板块 → 标记为"主线候选"
    └─ 若连续3周出现在主线候选中 → 升级为"确认主线"

Step 5: 龙头挖掘
    └─ 对每个确认主线，取板块内近20日涨幅前5的成分股
    └─ 标注：连板高度、市值、是否机构票/游资票
```

```markdown
## 四、历史主线持续性跟踪

```
